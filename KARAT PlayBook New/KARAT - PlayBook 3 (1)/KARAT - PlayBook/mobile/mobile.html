Mock Questions

Practice questions for mobile engineer interviews, organized by experience level.

Mid-Level Questions

1

Design a mobile app architecture for offline support.

Answer: Use local storage/SQLite with sync queue, conflict resolution strategy, background sync, and optimistic UI updates with eventual consistency.
2

How do you optimize app startup time?

Answer: Defer non-critical work, use lazy loading, reduce main thread work, optimize native module init, and use splash screen with quick first render.
3

Implement secure storage of user credentials.

Answer: Use platform secure stores (Keychain/Keystore), encrypt sensitive data, avoid storing raw tokens, and rotate secrets when needed.
4

How do you handle API errors gracefully?

Answer: Use Retrofit/URLSession with error callbacks, show user-friendly messages, and retry logic for transient errors.
5

Explain lazy loading in mobile apps.

Answer: Load resources or data only when needed, typically in response to user actions, to reduce initial load time and memory usage.
6

How do you optimize RecyclerView performance?

Answer: Use ViewHolder pattern, DiffUtil for updates, and avoid nested layouts.
7

What is the role of LiveData/Combine?

Answer: LiveData (Android) and Combine (iOS) are used for reactive programming, allowing UI components to observe data changes and update automatically.
8

How do you secure API keys in mobile apps?

Answer: Store API keys in secure storage (Keychain/Keystore), use environment variables, and avoid hardcoding keys in the source code.
9

Explain the difference between synchronous and asynchronous calls.

Answer: Synchronous calls block the executing thread until completion, while asynchronous calls allow the thread to continue executing and handle the result later.
10

How do you implement pagination in a list view?

Answer: Load data in chunks or pages, typically using a scroll listener to detect when to load the next page, and update the list adapter.
11

Write a simple Swift/Kotlin function for data binding.

Answer: Use property observers (didSet in Swift, set in Kotlin) to update the UI when the data changes.
12

How do you test UI components?

Answer: Use XCTest (iOS) or Espresso (Android) for UI testing, simulating user interactions and verifying UI behavior.
13

Explain the concept of View Binding.

Answer: View Binding is a feature that allows you to bind UI components in your layout to data sources in your app using a declarative format.
14

How do you handle configuration changes in Android?

Answer: Use ViewModel to retain UI state, and declare configuration changes in the manifest to prevent activity recreation.
15

What is the difference between fragments and activities?

Answer: Activities are the entry points for an app's interaction with the user, while fragments are reusable components with their own lifecycle, typically used within activities.
16

How do you implement caching for API responses?

Answer: Use caching libraries (e.g., Retrofit's caching, Room database) to store API responses and serve them from the cache when available.
17

Explain the role of dependency injection.

Answer: Dependency injection is a design pattern used to implement IoC (Inversion of Control), allowing dependencies to be provided to a class rather than the class creating them.
18

How do you manage app permissions?

Answer: Request permissions at runtime (Android) or declare them in the Info.plist (iOS), and handle the user's response to the permission request.
19

What is the difference between REST and GraphQL?

Answer: REST is an architectural style for designing networked applications, while GraphQL is a query language for APIs that allows clients to request only the data they need.
20

How do you implement localization?

Answer: Use platform localization features (e.g., Android's resources, iOS's .strings files) to provide translations and locale-specific resources.

Senior-Level Questions

1

Design an offline-first mobile architecture.

Answer: Local database with sync queue, conflict-resolution strategies, background sync, efficient change sets, and graceful degradation of features offline.
2

How do you ensure mobile app security?

Answer: Use secure storage, obfuscation, HTTPS, certificate pinning when appropriate, server-side validation, and secure APIs with least privilege.
3

How to optimize battery and network usage?

Answer: Batch network calls, use push for updates, avoid wakelocks, schedule background work, and use efficient codecs and compression.
4

Explain architecture for feature flagging in mobile apps.

Answer: Use remote config with local defaults, evaluate flags at startup and cache decisions, and support dynamic updates with safe fallbacks and rollbacks.
5

How to handle migrations for local DB schemas?

Answer: Use migration scripts with versioning, run migrations on app upgrade safely with fallback, and test migrations on large datasets.
6

How do you measure and improve app startup time?

Answer: Measure cold/warm starts, defer heavy work, use background initialization, optimize rendering, and reduce main thread work and initialization dependencies.
7

Design strategies for A/B testing in mobile apps.

Answer: Use server-side or client-side flagging, randomization and bucketing, collect metrics reliably even offline, and manage experiment lifecycle centrally.
8

Explain architecture for cross-platform apps (React Native/Flutter).

Answer: Share UI/business logic where possible, isolate native modules, ensure consistent testing across platforms, and address platform-specific UX nuances.
9

How to handle large media uploads reliably on mobile?

Answer: Use resumable uploads, chunking, pre-signed URLs, background uploads, and retry strategies with network awareness.
10

How do you instrument mobile apps for observability?

Answer: Collect crash reports, RUM metrics, custom events, breadcrumbs, and correlate with backend traces for end-to-end debugging.
11

How to manage app releases and rollbacks?

Answer: Use staged rollouts, feature flags for risky features, monitor crashes and metrics, and provide fast rollback paths via server-side flags or new builds.
12

How to design push notification architecture?

Answer: Use platform push services with targeted topics, dedupe and throttle notifications, and provide opt-in controls and personalization server-side.
13

Explain continuous integration and testing for mobile apps.

Answer: Automate builds, unit and integration tests, run on device farms or emulators, and gate releases with automated test suites and manual QA as needed.
14

How do you design for accessibility on mobile?

Answer: Support TalkBack/VoiceOver, provide semantic labels, ensure touch targets and contrast, and test with accessibility tools and users.
15

How to reduce app binary size?

Answer: Remove unused resources, use app bundles, enable code shrinking/obfuscation, and modularize features to download on demand.
16

Describe secure patterns for OAuth in mobile apps.

Answer: Use PKCE flows, avoid embedding client secrets, use refresh token rotation, validate tokens on server, and secure redirect URIs.
17

How to handle multi-process architectures on Android?

Answer: Carefully design IPC via bound services or content providers, manage shared resources, and test lifecycle interactions to avoid race conditions.
18

How do you perform regression testing for UI changes?

Answer: Use snapshot/UI tests, run on multiple devices, complement with manual exploratory testing, and prioritize critical flows for automation.
19

How to manage user data privacy on mobile?

Answer: Minimize collection, store sensitive data encrypted, implement consent flows, and provide clear privacy controls and retention policies.
20

How to scale backend to support millions of mobile clients?

Answer: Use stateless APIs with autoscaling, efficient caching, CDNs for assets, and partitioning/sharding for user data with robust rate limiting and monitoring.
