Mock Questions

Practice questions for architect interviews, organized by category.

System Design Questions

1

Design a scalable URL shortening service (like bit.ly).

Answer: Use short key generation, distributed datastore for mappings, cache hot entries, handle collisions, and add analytics and rate limiting for abuse protection.
2

How would you architect a real-time chat application?

Answer: Use WebSockets with load-balanced connection brokers, message queues for delivery, persistent storage for history, presence service, and partition by channels for scale.
3

What are the trade-offs between monolithic and microservices architectures?

Answer: Monoliths simplify deployment and testing but limit scale and team autonomy; microservices enable independent deploys and scale but add complexity (operational overhead, distributed transactions).
4

Design a system for processing large-scale data analytics.

Answer: Use distributed processing (Spark/Flink), partitioned storage (data lake), streaming ingestion (Kafka), and separate offline/online serving layers for queries and dashboards.
5

How would you ensure high availability for a critical web service?

Answer: Deploy multi-region replicas, use health checks and failover, redundancy at each layer (LBs, DB replicas), chaos testing, and robust monitoring and automated recovery.

Code Review Questions

1

What do you look for in a code review?

Answer: Correctness, clarity, test coverage, performance implications, security, and adherence to style and architecture guidelines.
2

How do you give constructive feedback?

Answer: Be specific, explain impact, suggest alternatives, focus on code not person, and acknowledge good parts.
3

How do you evaluate test quality?

Answer: Check that tests cover edge cases, assert behavior not implementation, are deterministic, and validate error paths and performance where needed.
4

What are common performance anti-patterns to spot?

Answer: N+1 queries, unnecessary blocking calls, repeated expensive computations, and unbounded memory growth.
5

How do you assess security in a code review?

Answer: Look for input validation, proper encoding/escaping, safe use of secrets, secure defaults, and dependency vulnerabilities.

Scenario 1: System Design Discussion

Interviewer:

Design an online file storage system (like Dropbox).

Candidate:

I'd start by clarifying requirements: expected scale, file size limits, sharing features, and security needs. At a high level, I'd use a microservices architecture with separate services for file upload/download, metadata management, and user authentication. Files would be stored in object storage, with metadata in a relational database. I'd use a CDN for fast file delivery and implement redundancy for reliability.
