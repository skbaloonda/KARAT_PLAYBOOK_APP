Introduction

Welcome to the Data Analyst Playbook! This comprehensive guide will help you master data querying, analysis, visualization, and business intelligence essential for turning data into actionable insights.

Core Responsibilities

  • Data Querying: Write efficient SQL queries for data extraction, transformation, and reporting
  • Data Cleaning & Preparation: Use tools like Excel, Python (Pandas), or SQL to clean and preprocess data
  • Data Modeling: Design schemas, normalize data, and create relationships between tables
  • Analysis & Visualization: Use BI tools (Power BI, Tableau) and statistical methods to analyze and visualize data
  • Business Communication: Present findings to stakeholders in clear, actionable formats

Practice Questions

Comprehensive set of practice questions for data analyst interviews, organized by experience level.

Mid-Level Questions

1

Write a SQL query to find the top 5 customers by sales.

Answer: Use GROUP BY customer, SUM(sales), ORDER BY total DESC LIMIT 5. Ensure proper indexes on customer and date columns.
2

Design an ER diagram for an e-commerce platform.

Answer: Entities: Users, Orders, Products, OrderItems, Payments; define PK/FK relationships and include attributes like timestamps and status.
3

How would you clean and preprocess a messy dataset?

Answer: Inspect data, handle missing values (impute/drop), normalize types, deduplicate, standardize formats, and log transformations for reproducibility.
4

Build a simple ETL pipeline for daily sales data.

Answer: Extract from source, transform (clean, aggregate), load into data warehouse, schedule via Airflow, and validate with checksums and alerts.
5

Write a query to calculate monthly sales totals.

Answer: GROUP BY YEAR(date), MONTH(date) with SUM(sales); use date_trunc or EXTRACT depending on DB, and index date column for speed.
6

How do you handle missing values in data?

Answer: Analyze missingness, impute with mean/median/mode or model-based methods, use flags, or drop rows/columns if appropriate.
7

Design a dashboard for product performance.

Answer: Include KPIs (sales, conversion, returns), filters (time, region), trend charts, and drilldowns; ensure refresh cadence and access controls.
8

Write a query to join customer and order tables.

Answer: Use SELECT c.*, o.* FROM customers c JOIN orders o ON c.id = o.customer_id; choose INNER/LEFT depending on requirement.
9

How do you validate data quality in ETL?

Answer: Implement checks (row counts, null rates, referential integrity), compare source/target stats, and alert on anomalies.
10

Create a report showing sales by region.

Answer: Aggregate sales by region using GROUP BY, visualize with bar or map chart, and provide filter controls for time and product.
11

Write a query to find duplicate records.

Answer: Use GROUP BY columns HAVING COUNT(*) > 1 to identify duplicates, then inspect and decide dedup strategy.
12

How do you optimize a slow-running query?

Answer: Analyze EXPLAIN plan, add indexes, rewrite joins/subqueries, limit columns, and consider denormalization or partitioning.
13

Design a normalized schema for a CRM system.

Answer: Separate contacts, accounts, interactions, opportunities into linked tables with appropriate foreign keys to reduce redundancy.
14

Write a query to count active users per day.

Answer: GROUP BY DATE(activity_timestamp) with COUNT(DISTINCT user_id), and filter by activity types that indicate active usage.
15

How do you secure sensitive data in reports?

Answer: Mask or aggregate PII, enforce RBAC, encrypt data at rest and in transit, and audit report access.
16

Build a simple data dictionary for a project.

Answer: Document table/field definitions, data types, owners, update frequency and example values; store centrally and version it.
17

Write a query to aggregate sales by product category.

Answer: JOIN products to sales, GROUP BY category_id, SUM(sales_amount), and order by total desc.
18

How do you document data lineage?

Answer: Track source systems, transformation steps, jobs and schedules, and produce visual lineage diagrams tied to pipeline metadata.
19

Design a role-based access control system for analytics.

Answer: Define roles and permissions, map users to roles, enforce at report and dataset level, and audit permission changes.
20

Write a query to filter out outliers in sales data.

Answer: Use statistical methods (IQR or z-score) to detect and filter values beyond thresholds, or cap extreme values for robustness.

Senior-Level Questions

1

Design an analytics data model for tracking user behavior.

Answer: Use event-based raw ingestion, canonical user and event tables, partition by time, and build derived aggregates for common queries; track user identifiers and session metadata.
2

How do you handle slowly changing dimensions?

Answer: Use SCD Type 2 with effective dates to preserve history, or Type 1 for overwrite where history isn't required; store versioned keys and audit info.
3

Explain partitioning strategies for large fact tables.

Answer: Partition by date for time-series, use range or hash partitions for scale, and ensure queries use partition keys to prune data.
4

Design a data pipeline for near-real-time analytics.

Answer: Use streaming ingestion (Kafka), stream processing (Kafka Streams/Flink), materialize views to serving stores, and monitor latency with SLA alerts.
5

How to ensure data quality in reporting?

Answer: Add checks at ingestion, schema validation, row count reconciliation, anomaly detection, and automated alerts on data drift.
6

Explain dimensional modeling best practices.

Answer: Use star schemas for analytics, keep dimensions denormalized for easy querying, and design grain clearly with surrogate keys.
7

How do you optimize slow analytical queries?

Answer: Add appropriate indexes/materialized views, rewrite queries to reduce data scanned, partition/prune, and leverage columnar storage.
8

Design a dashboarding strategy for executives vs. analysts.

Answer: Executives: KPIs and high-level trends; analysts: interactive drill-downs and raw data access; choose caching strategies and access controls accordingly.
9

How to manage data lineage and auditing?

Answer: Capture metadata about pipelines, store transformations, track dataset versions, and expose lineage via automated tools for impact analysis.
10

Discuss feature store design for ML features.

Answer: Centralized feature registry, consistent offline/online feature computations, time-travel or point-in-time correctness, and monitoring feature drift.
11

How do you balance freshness vs cost in analytics?

Answer: Categorize datasets by SLA, use micro-batching for medium freshness, streaming for critical paths, and cache less critical aggregates to save cost.
12

Explain row-level security implementation options.

Answer: Enforce in query layer via app logic, use DB policies (e.g., Postgres RLS), or expose filtered views per role; prefer DB-native controls for simplicity.
13

How to perform A/B analysis with noisy data?

Answer: Use randomization, ensure sufficiently powered samples, use robust metrics, and apply statistical techniques (bootstrap, confidence intervals) to account for noise.
14

Design a reusable ETL framework.

Answer: Modular extract/transform/load stages, standardized connectors, metadata-driven jobs, idempotency, and centralized monitoring and retries.
15

How to version datasets for reproducibility?

Answer: Snapshot datasets, record transformation code and parameters, store hashes and metadata, and use data cataloging with versions.
16

Explain trade-offs between OLAP and data lake architectures.

Answer: OLAP offers fast structured queries with pre-aggregation; data lakes offer flexibility and cost-effective storage but require more processing for analysis.
17

How do you monitor model feature drift?

Answer: Track feature distributions over time, set thresholds/alerts, and retrain models or recalibrate when drift impacts performance.
18

Describe strategies for data anonymization.

Answer: Masking, tokenization, aggregation, k-anonymity, differential privacy approaches depending on sensitivity and use cases.
19

How to design cost-effective archival strategies?

Answer: Move cold data to cheaper storage tiers, compress and partition, keep metadata for discoverability, and define retention policies aligned to compliance.
20

How to onboard a new data domain into your platform?

Answer: Define schema and ownership, create ingestion jobs with validation, document lineage, apply access controls, and run integration tests with consumers.

Mid-Level 3 Mock Interview Scripts

Scenario 1: SQL Query for Top Customers
Interviewer:
Write a SQL query to find the top 5 customers by sales.
Candidate:
SELECT customer_id, SUM(sales) AS total_sales FROM orders GROUP BY customer_id ORDER BY total_sales DESC LIMIT 5;
Interviewer:
How would you handle ties?
Candidate:
Use RANK() window function to rank customers and filter where rank <= 5.
Scenario 2: Data Cleaning in ETL
Interviewer:
How would you clean and preprocess a messy dataset?
Candidate:
Remove duplicates, handle missing values, standardize formats, and validate data types using SQL and Python (Pandas).
Interviewer:
How do you automate this?
Candidate:
Build ETL scripts with scheduled jobs and error logging.
Scenario 3: Dashboard Design
Interviewer:
Design a dashboard for product performance.
Candidate:
Include KPIs like sales, returns, and ratings. Use filters for time and category. Visualize with bar charts and trend lines.
Interviewer:
How do you ensure data accuracy?
Candidate:
Validate source data and refresh dashboards on schedule.

Senior-Level 3 Mock Interview Scripts

Scenario 1: Advanced SQL for Running Totals
Interviewer:
Write a SQL query using window functions for running totals.
Candidate:
SELECT date, sales, SUM(sales) OVER (ORDER BY date) AS running_total FROM sales_data;
Interviewer:
How do you optimize this for large datasets?
Candidate:
Use partitioning and indexing on the date column.
Scenario 2: Scalable ETL Architecture
Interviewer:
Architect a scalable ETL pipeline for multi-source data.
Candidate:
Use modular ETL steps (extract, transform, load), schedule with Airflow, and store intermediate results in staging tables.
Interviewer:
How do you handle schema changes?
Candidate:
Implement schema versioning and automated migration scripts.
Scenario 3: Data Governance and Compliance
Interviewer:
Design a data governance framework for analytics.
Candidate:
Define data ownership, access policies, and audit trails. Use tools for lineage tracking and compliance monitoring.
Interviewer:
How do you ensure GDPR compliance?
Candidate:
Mask PII, log data access, and enable user data deletion.
