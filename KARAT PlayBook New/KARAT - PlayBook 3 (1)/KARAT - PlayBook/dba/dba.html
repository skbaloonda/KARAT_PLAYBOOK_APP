Mock Questions

Practice questions for DBA interviews, organized by experience level.

Mid-Level Questions (GCB5)

1

Write a SQL query to find duplicate records in a table.

Answer: Use GROUP BY columns HAVING COUNT(*)>1 to identify duplicates, then select rows to inspect or delete based on ROW_NUMBER() over partition for de-duplication.
2

Explain normalization and its benefits.

Answer: Normalization reduces redundancy and update anomalies by organizing data into related tables (1NF,2NF,3NF...), improving consistency and maintainability.
3

How would you back up a database? Describe the steps.

Answer: Schedule full and incremental backups, validate backup integrity, store offsite, automate retention, and test restores periodically.
4

What is an index? How does it improve performance?

Answer: An index is a data structure speeding lookups by reducing scanned rows; it improves query performance but adds write overhead and storage cost.
5

Troubleshoot a scenario where a query is running slowly.

Answer: Check EXPLAIN plan, look for full table scans, missing indexes, high I/O, costly joins; add indexes, rewrite query, or optimize schema and statistics.

Senior-Level Questions

1

Design a high-availability database architecture.

Answer: Use primary-replica replication, automated failover, synchronous/asynchronous replication trade-offs, and multi-zone deployment for redundancy.
2

How do you tune database performance under heavy load?

Answer: Analyze queries/indices, add appropriate indexes, optimize schema, tune buffer pools, and use read replicas and caching for read-heavy workloads.
3

Explain strategies for online schema migrations.

Answer: Use backward-compatible changes, online migration tools, rolling schema updates, and deploy application changes that support both schemas during transition.
4

How do you investigate and resolve deadlocks?

Answer: Capture deadlock graphs, analyze conflicting queries, reduce lock scope, add appropriate indexes, and retry transactions with backoff when necessary.
5

Design backup and restore with minimal RTO.

Answer: Use incremental backups, point-in-time recovery, warm standby replicas, and automate restore runbooks and regular restore drills.
6

How to plan capacity for transactional databases?

Answer: Model transaction volume, peak concurrency, I/O requirements, and provision headroom with autoscaling and read/write separation.
7

Explain partitioning strategies for OLTP workloads.

Answer: Use range/hash partitioning to improve query pruning and manage data growth, and ensure application uses partition keys for performance.
8

How do you secure database access?

Answer: Use least-privilege accounts, network restrictions, encrypted connections, role-based access, and audit logging for sensitive operations.
9

How to manage large data purges without impacting performance?

Answer: Batch deletes, use TTL or partition drop, throttle operations, and run during maintenance windows while monitoring impact.
10

Discuss strategies for cross-database replication.

Answer: Use CDC-based replication, ensure schema mapping, handle conflict resolution, and monitor replication lag and consistency.
11

How do you enforce data integrity at scale?

Answer: Use constraints where possible, application-level checks, referential integrity, and background audits for eventual consistency issues.
12

Explain database auditing and compliance practices.

Answer: Capture change logs, retain access logs securely, implement role-based access, and automate reports required by regulators.
13

How do you approach index maintenance in large tables?

Answer: Schedule rebuilds during low-traffic windows, monitor fragmentation, and balance speed of writes with need for query performance.
14

How to troubleshoot slow queries that vary over time?

Answer: Compare plans over time, check stats and parameter sniffing, review data distribution changes, and monitor resource contention and hardware metrics.
15

Explain multi-master replication challenges and solutions.

Answer: Handle conflict resolution, ensure unique keys, accept eventual consistency, and use application-level reconciliation or CRDTs where appropriate.
16

How to perform capacity testing for databases?

Answer: Simulate realistic workloads, ramp concurrency, measure latencies and resource usage, and validate scaling strategies against targets.
17

How do you manage multi-tenant data in a shared database?

Answer: Use tenant identifiers, row-level isolation, resource quotas, and careful indexing and security to prevent cross-tenant access.
18

Discuss backup encryption and secure storage best practices.

Answer: Encrypt backups at rest, rotate keys, store offsite with access controls, and validate backups regularly.
19

How to implement failover testing and drills?

Answer: Run planned failovers in staging and production with runbooks, validate application behavior, and measure RTO/RPO to improve procedures.
20

How do you incorporate automation in DBA tasks?

Answer: Automate backups, monitoring, alerting, index maintenance, and standard operational tasks via scripts and orchestration tools to reduce human error.

Scenario 1: Troubleshooting Slow Query

Interviewer:

A user reports that a report query is taking too long to run. How would you approach this?

Candidate:

I'd start by examining the query execution plan, checking for missing indexes, and reviewing server resource usage. I'd also look for blocking or locking issues and consider rewriting the query for efficiency.

Scenario 2: Advanced SQL

Interviewer:

Write a query to rank employees by salary within each department.

Candidate:

I'd use the RANK() window function partitioned by department and ordered by salary descending.

SELECT 
    employee_id,
    employee_name,
    department,
    salary,
    RANK() OVER (
        PARTITION BY department 
        ORDER BY salary DESC
    ) AS salary_rank
FROM employees;
