Mock Questions

Practice questions for mainframe COBOL engineer interviews (Mid-Level - GCB5).

Priority Questions

1

Transaction file sequential read and account file read random update for account balances based on transaction file input.

Answer: Read transaction file sequentially, lookup account records via indexed DB or VSAM, apply updates in transactional batch, and checkpoint to allow restart on failure.
2

JCL was given and in that JCL one program execution step was mentioned along with input transaction file and output account file. In second step transaction file delete using IDCAMS.

Answer: Ensure correct dataset allocation, proper DISP settings, validate IDCAMS control statements, and sequence steps with return code checks to avoid data loss.
3

One entire program was mentioned with read and write file; environment section was missing.

Answer: Add environment config (DD statements, dataset attributes), verify JCL parameters, and ensure correct file modes and dataset disposition.
4

In production if a system patch is applied and the same day batch started abending. How will you fix it and what steps will you take so that it does NOT happen in the future?

Answer: Roll back patch if needed, analyze job logs, reproduce in staging, add regression tests, freeze deployments during critical batches, and improve rollout plan with validation windows.
5

DB2 program is giving an error -911; how will you find the error and resolve the issue?

Answer: Check DB2 SQLCA/SQLCODE, review lock contention and transaction logs, retry logic for deadlocks, and tune isolation levels or application transaction boundaries.
6

How will you optimize DB2 performance?

Answer: Analyze access paths, add/rebuild indexes, update statistics, tune buffer pools, and optimize SQL and I/O patterns.
7

In SIT environment a job started failing and the remaining jobs should complete early; what steps will you take to finish the batch and prevent this from happening again in the test environment?

8

One job abended with a space issue; how will you investigate and rectify it?

Mid-Level Questions

1

Explain VSAM file organization and use cases.

Answer: VSAM provides keyed and sequential access (KSDS, ESDS, RRDS) for high-performance mainframe file operations; used for indexed datasets and batch processing.
2

How do you tune JCL jobs for performance?

Answer: Optimize dataset layouts, allocate appropriate buffers, parallelize steps when safe, and profile I/O and CPU to identify bottlenecks.
3

Describe common IDCAMS utilities and their purposes.

Answer: IDCAMS manages VSAM datasets: DEFINE, REPRO, DELETE, REORG and is used for dataset lifecycle and maintenance tasks.
4

How to troubleshoot abends and analyze dump outputs?

Answer: Review SVC dump, system logs, return codes, and check program logs; reproduce in test and apply fixes for invalid data or resource issues.
5

Explain DB2 locking and concurrency controls.

Answer: DB2 uses lock modes and isolation levels; tune for concurrency by adjusting isolation and using row-level locking and proper indexing.
6

How do you plan batch windows and job dependencies?

Answer: Map job dependencies, schedule critical jobs in safe windows, use workload managers, and implement checkpoints to resume after failures.
7

Describe techniques for mainframe performance monitoring.

Answer: Monitor CPU, I/O, spool, DASD metrics, track DASD response time, check job runtimes and DB2 stats for proactive tuning.
8

How to manage dataset backups and restores?

Answer: Use system-level backup tools, tape/archive processes, validate backups, and document restore procedures for quick recovery.
9

Explain CICS transaction processing basics.

Answer: CICS handles online transactional workloads with units of work, transient data, and integrates with DB2 and MQ for backend operations.
10

How do you handle high-volume file processing?

Answer: Use efficient I/O, buffered reads, parallel processing where safe, and optimize dataset extents and block sizes for throughput.
11

Explain JES spool management best practices.

Answer: Monitor spool usage, purge old output, tune retention policies, and manage output classes to prevent spool exhaustion.
12

How to apply system patches without disrupting nightly batches?

Answer: Stagger patch windows, test in staging, have rollback plans, and schedule non-disruptive components first with validation steps.
13

How to integrate mainframe systems with modern services?

Answer: Use middleware (MQ, APIs), wrap legacy programs with service layers, and use adapters to translate protocols and data formats.
14

How to handle restartability for long-running jobs?

Answer: Implement checkpoints, idempotent processing, and design jobs to resume from last successful checkpoint with consistent inputs.
15

Describe dataset organization strategies for performance.

Answer: Choose appropriate record formats and block sizes, use partitioning and clustering where available, and align extents to I/O patterns.
16

How to diagnose DB2 performance regressions?

Answer: Compare access paths, check buffer pool stats, review recent changes to SQL and indexes, and examine system resource metrics.
17

Explain best practices for JCL parameterization and reuse.

Answer: Use PROC with symbolic parameters, keep JCL modular, and maintain library of tested procs for reuse and consistency.
18

How to handle system capacity planning on mainframes?

Answer: Analyze historical job volumes, CPU and I/O usage, project growth, and plan LPAR and DASD capacity with buffer for peaks.
19

Describe approaches for automation in mainframe operations.

Answer: Automate job scheduling, monitoring and alerting, use scriptable admin tools and integrate with orchestration platforms for cross-platform workflows.
20

How do you ensure secure access and audit on mainframes?

Answer: Enforce RACF or equivalent, implement least privilege, enable detailed audit logging, and review access periodically.

Scenario 1: JCL Job Restart

Interviewer:

How do you restart a failed JCL job?

Candidate:

Use the RESTART parameter in the JOB statement and specify the step name. Ensure checkpoint logic in COBOL for data consistency.

//JOBNAME JOB ...,RESTART=STEPNAME
//STEP1 EXEC PGM=COBOLPGM
//SYSOUT DD SYSOUT=*
Scenario 2: COBOL File Processing

Interviewer:

Write a COBOL program to read a VSAM file and calculate totals.

Candidate:

Here's the approach:

  • OPEN INPUT file
  • READ record
  • ADD amount to total
  • WRITE output record
  • CLOSE file

Complexity: O(n) for n records.

Scenario 3: DB2 Query Optimization

Interviewer:

How do you optimize a DB2 query?

Candidate:

Use indexed columns, avoid unnecessary joins, and analyze EXPLAIN plan for performance tuning.

-- Create index for better performance
CREATE INDEX IDX_CUSTOMER 
ON CUSTOMERS(CUSTOMER_ID);

-- Optimized query
SELECT * FROM CUSTOMERS 
WHERE CUSTOMER_ID = 12345;
