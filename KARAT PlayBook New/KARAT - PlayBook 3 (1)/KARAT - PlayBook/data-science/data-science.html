Practice Questions

Comprehensive set of practice questions for data science interviews, organized by experience level.

Mid-Level Questions

1

Explain the bias-variance tradeoff in machine learning.

Answer: Bias is error from wrong assumptions (underfitting); variance is error from sensitivity to training data (overfitting). Balance via model complexity and regularization.
2

How would you handle missing data in a dataset?

Answer: Analyze missingness, impute (mean/median/model), use flags, or drop rows/columns depending on impact and mechanism (MCAR/MAR/MNAR).
3

Write a Python function to normalize numerical features.

Answer: Use sklearn.preprocessing.StandardScaler or implement (x - mean)/std to standardize features.
4

Explain the difference between supervised and unsupervised learning.

Answer: Supervised uses labeled data to predict targets; unsupervised finds structure in unlabeled data (clustering, dimensionality reduction).
5

What is regularization, and why is it important?

Answer: Regularization (L1/L2) penalizes large weights to reduce overfitting and improve generalization.
6

How do you evaluate a classification model's performance?

Answer: Use metrics like accuracy, precision, recall, F1, ROC-AUC and confusion matrix; choose based on class imbalance and business needs.
7

Implement a simple linear regression using NumPy.

Answer: Use normal equation: w = (X^T X)^{-1} X^T y or use gradient descent; prefer libraries for numerical stability.
8

What are decision trees, and how do they work?

Answer: Trees split data on features to create homogeneous leaves; use criteria like Gini/entropy, control depth to avoid overfitting, and prune if needed.
9

Explain cross-validation and its importance.

Answer: Cross-validation (k-fold) estimates model generalization by training/validating across folds and reduces variance from single train/test splits.
10

How would you detect outliers in a dataset?

Answer: Use statistical methods (z-score, IQR), visualization (boxplots), or model-based detection (isolation forest) depending on context.
11

What is feature engineering, and why is it critical?

Answer: Creating informative features increases model signal; includes encoding, aggregation, interactions, and domain-specific transforms.
12

Implement a k-means clustering algorithm.

Answer: Initialize centroids, assign points to nearest centroid, recompute centroids, iterate until convergence; use kmeans++ init and scale features.
13

Explain precision, recall, and F1-score.

Answer: Precision = TP/(TP+FP), Recall = TP/(TP+FN); F1 is harmonic mean of precision and recall, useful for imbalanced classes.
14

What are hyperparameters, and how do you tune them?

Answer: Hyperparameters control learning (e.g., regularization, tree depth); tune via grid/random search or Bayesian optimization with cross-validation.
15

How do you handle imbalanced datasets?

Answer: Use resampling (oversample/undersample), class weighting, appropriate metrics (precision/recall), or specialized algorithms like SMOTE.
16

Explain logistic regression vs. linear regression.

Answer: Linear regression predicts continuous targets; logistic regression models probability for binary classification using sigmoid and log-loss.
17

What is gradient descent, and how does it work?

Answer: Optimization algorithm that iteratively updates parameters opposite to gradient of loss; use learning rate, batch variants, and momentum for stability.
18

Write code to perform one-hot encoding in Pandas.

Answer: Use pd.get_dummies(df, columns=['column_name']) to convert categorical variable into dummy/indicator variables.
19

What is the ROC curve, and how is it used?

Answer: ROC curve plots true positive rate vs. false positive rate at various thresholds; used to select optimal model and discard suboptimal ones.
20

Explain the concept of overfitting and how to prevent it.

Answer: Overfitting is modeling noise instead of signal; prevent with regularization, pruning, dropout, and by ensuring sufficient training data.

Senior-Level Questions

1

Design an end-to-end ML system for churn prediction.

Answer: Define data sources, feature engineering pipelines, train/validate models, deploy with model versioning, monitor drift, and integrate with business actions and feedback loop.
2

How do you validate model performance in production?

Answer: Monitor key metrics (AUC, precision/recall), track data distribution drift, use shadow testing, and compare against baseline and business KPIs.
3

Explain techniques to reduce model latency.

Answer: Use model compression, feature precomputation, caching, optimized inference runtimes (ONNX/TensorRT), and horizontal scaling of inference services.
4

How do you handle class imbalance?

Answer: Use resampling, class weighting, appropriate metrics, ensemble methods, or anomaly detection approaches depending on severity.
5

Design experiments to evaluate feature importance.

Answer: Use permutation importance, SHAP/ICE plots, ablation studies and controlled experiments to measure the impact on model metrics and business outcomes.
6

Explain bias and fairness checks in models.

Answer: Measure metrics across groups, use fairness-aware algorithms, apply preprocessing/postprocessing fixes, and document trade-offs and mitigation strategies.
7

How to implement feature stores for online inference?

Answer: Serve low-latency feature lookups via key-value store, ensure consistency with offline features, and maintain ingestion pipelines and monitoring.
8

How to detect and respond to data drift?

Answer: Monitor statistical tests on feature distributions, set thresholds, alert teams, and retrain or reweight models when drift impacts performance.
9

Explain approaches for model explainability.

Answer: Use global and local explainers (SHAP, LIME), feature importance, counterfactuals, and communicate explanations in business terms.
10

How to design A/B tests for ML-driven features?

Answer: Randomize users, define clear metrics, ensure guardrail metrics, run for statistically valid periods, and monitor for novelty effects.
11

How do you perform feature selection at scale?

Answer: Use automated methods (L1, tree-based importances), correlation filtering, and domain-driven pruning with cross-validation to validate impact.
12

Explain trade-offs between batch and online learning.

Answer: Batch learning is stable and uses full datasets; online adapts quickly to new data but may be noisy and requires careful learning-rate and drift handling.
13

How do you version models and datasets?

Answer: Use model registry with metadata, tag datasets with hashes and storage locations, and record training parameters and environment for reproducibility.
14

How to evaluate causal impact rather than correlation?

Answer: Use randomized experiments, instrumental variables, difference-in-differences, or causal inference frameworks to identify causal effects.
15

How do you monitor model fairness over time?

Answer: Track fairness metrics across subgroups, set alerts for drift, and include governance to review and remediate issues when thresholds trip.
16

How to handle missing features in production inference?

Answer: Use default values, imputation strategies, or fallback models and track missingness patterns to avoid biased predictions.
17

Explain strategies for model ensemble deployment.

Answer: Use weighted ensembles or stacked models, manage inference cost by gating expensive models, and ensure consistent feature pipelines.
18

How to debug production inference failures?

Answer: Collect inputs/outputs, compare to training distribution, check feature pipeline health, and use replay tests and logs to reproduce and fix.
19

What are best practices for ML model monitoring?

Answer: Monitor input distributions, prediction quality, latency, resource usage, and set alerts with dashboards and automated retraining triggers.
20

How do you quantify business value for ML projects?

Answer: Map model outputs to business KPIs, run experiments to measure lift, estimate ROI including infra and maintenance costs, and prioritize features by impact.

Scenario 1: SQL Aggregation

Interviewer:

Write a SQL query to calculate total sales per region.

Candidate:

I'll use GROUP BY to aggregate sales by region:

SELECT region, SUM(sales) AS total_sales
FROM orders
GROUP BY region
ORDER BY total_sales DESC;

This query groups all orders by region and calculates the sum of sales for each region, ordered from highest to lowest.

Scenario 2: Data Cleaning in Python

Interviewer:

How do you handle missing values in Python?

Candidate:

There are several approaches depending on the context:

  • Use df.fillna(value) to fill missing values with a specific value
  • Use df.fillna(df.mean()) for numerical columns to fill with mean
  • Use df.dropna() to remove rows with missing values
  • Use df.interpolate() for time-series data

The choice depends on the data type, amount of missing data, and business requirements.

Scenario 3: Visualization

Interviewer:

Create a bar chart showing sales by category.

Candidate:

I'll use Matplotlib to create the visualization:

import matplotlib.pyplot as plt

# Group by category and sum sales
category_sales = df.groupby('category')['sales'].sum()

# Create bar chart
category_sales.plot(kind='bar', color='steelblue')
plt.title('Sales by Category')
plt.xlabel('Category')
plt.ylabel('Total Sales')
plt.xticks(rotation=45)
plt.tight_layout()
plt.show()

This creates a clear bar chart with proper labels and formatting.

Scenario 1: Advanced SQL with Window Functions

Interviewer:

Write a SQL query using window functions for ranking customers by sales.

Candidate:

I'll use the RANK() window function to rank customers:

SELECT 
    customer_id,
    customer_name,
    SUM(sales) AS total_sales,
    RANK() OVER (ORDER BY SUM(sales) DESC) AS sales_rank
FROM orders
GROUP BY customer_id, customer_name
ORDER BY sales_rank;

This query calculates total sales per customer and assigns a rank based on sales. RANK() handles ties by giving them the same rank, while the next rank skips accordingly.

Scenario 2: ETL Pipeline Design

Interviewer:

Design a scalable ETL pipeline in Python.

Candidate:

I would design a modular ETL pipeline with these components:

  • Extract: Read data from source (API, database, files) using appropriate connectors
  • Transform: Use Pandas for data cleaning, validation, and feature engineering
  • Load: Write to target database or data warehouse using batch inserts

For scalability, I would:

  • Use chunking to process large datasets in batches
  • Implement error handling and logging at each step
  • Use Airflow or similar tool for scheduling and monitoring
  • Add data quality checks between stages
  • Implement retry logic for failed batches
Scenario 3: Feature Engineering

Interviewer:

How do you create features for a churn model?

Candidate:

For a churn prediction model, I would create these types of features:

  • Recency Features: Days since last purchase, last login, last interaction
  • Frequency Features: Number of purchases, login frequency, support tickets
  • Monetary Features: Total spend, average order value, lifetime value
  • Engagement Features: Time spent on platform, features used, content consumed
  • Trend Features: Change in activity over time (increasing/decreasing)
  • Categorical Encoding: One-hot encode or target encode categories

I would also create interaction features between variables and use domain knowledge to identify relevant patterns that might indicate churn risk.
