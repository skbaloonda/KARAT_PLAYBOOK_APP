Introduction to Roles/Personas

Category Persona GCB Level Description Test Composition
Engineering Frontend GCB4 The Frontend Engineer persona takes a framework-agnostic approach to assess frontend engineering abilities. This persona has two defined levels, GCB5 (Mid-level) and GCB4 (Senior). Mid-level allows candidates to select their preferred framework, React or Angular, to demonstrate experience working in modern frontend development. The Senior level evaluates frontend focused System Design and uses a single scenario to probe into many areas, including accessibility. Both levels share a frontend coding assessment, which requires improving a page layout, debugging, and adding functionality to an existing codebase in both HTML/CSS, and JavaScript. Frontend development focuses on UI/UX development languages and frameworks. It is intended for engineers that have focused their software skill around User Interfaces and customer experience. Expected to understand JavaScript, HTML, CSS, React / Angular frameworks. The GCB4 role has more emphasis on design capabilities. 20min - Frontend System Design - Depth - 50% 30min - Frontend Coding - 50%
Engineering Frontend GCB5 Similar to GCB4 10min - Frontend Frameworks - 40% 40min - Frontend Coding - 60%
Engineering Backend GCB4 Backend Engineer is the workhorse of the IT department. It's the archetype - solving problems with software base. Languages covered are: Python, Java, C++, C#, Clojure, CoffeeScript, F#, Go, Markdown, Ruby, Rust, TypeScript, Scala. The assessment consists of a hands-on coding task as well as a (virtual whiteboard) session with classic system design questions. The Backend Engineer persona takes a language-agnostic approach to assess backend engineering abilities. This persona has two defined levels, GCB5 (Mid-level) and GCB4 (Senior). Both levels evaluate System Design abilities, however, Mid-level assesses a series of isolated System Design topics, while Senior uses a single scenario to probe into many areas of System Design. Both levels share a Real World Coding assessment, which requires debugging, and adding functionality to an existing codebase in a preferred programming language. The GCB4 role has more emphasis on system design than GCB5, as the role often involves more solution design than the junior roles. 20min - System Design Depth - 50% 30min - Real World Backend Coding - 50%
Engineering Backend GCB5 Similar to GCB4 20min - System Design Breadth - 40% 30min - Real World Backend Coding - 60%

Engineering Mobile GCB4 The Mobile Engineer persona has two defined levels, GCB5 (Mid-level) and GCB4 (Senior), with Mid-level being split into iOS and Android variants. Both levels share a Real World coding assessment, which requires debugging and adding functionality to an existing codebase and is expected to be completed in a mobile development language such as Swift or Kotlin. At Mid-level the assessment includes a dedicated probe into accessible development through a presented scenario with accessibility flaws to identify, and continues to platform-specific (iOS and Android) discussion questions. At the senior level, accessibility is one component evaluated as part of an in-depth Whiteboard System Design exercise where designs are expected to function cross-platform. The GCB4 Mobile Developer assessment requires an understanding of the mobile landscape across iOS and Android development frameworks. 50% of the assessment is real world coding on a mobile platform. GCB5 engineering includes questions on accessibility and good design but is predominantly coding. The GCB4 coding is focused on either iOS and Android environments and engineers are not expected to have a broad understanding of the other platform. 30 min - Mobile System Design Whiteboard - 50% 20 min - Real World Mobile Coding - 50%
