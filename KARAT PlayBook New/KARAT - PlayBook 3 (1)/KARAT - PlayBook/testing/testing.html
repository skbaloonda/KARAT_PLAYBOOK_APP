Practice Questions

Comprehensive set of practice questions for quality engineering interviews, organized by experience level.

Priority Questions

1

What are the possible test cases for a credit card payments external API? Consider both positive and negative scenarios.

Answer: Positive: valid card, sufficient funds, accepted currency; Negative: invalid card, expired, insufficient funds, network timeout, declined by gateway, invalid params, concurrency failures.
2

What are the risks and considerations when using production data in a test environment?

Answer: PII leakage, compliance, environment parity, data freshness; mitigate by masking/anonymizing, access controls, and subset sampling.
3

Describe some bugs you have found during testing. How did you identify and resolve them?

Answer: Identify via failing test or logs, reproduce, isolate root cause, create bug ticket with repro steps, fix and add regression test, and verify in CI.
4

What is fuzz testing, and when is it useful?

Answer: Fuzzing sends random/ malformed input to find crashes and security issues; useful for parsers, protocols, and input validation testing.
5

Explain the bug lifecycle from discovery to resolution.

Answer: Discovery -> Triage -> Report -> Assign -> Fix -> Review -> Test -> Close; include severity, priority, and regression tests.
6

What are the key components of a good test automation framework?

Answer: Clear structure, reliable locators, data-driven tests, reporting, CI integration, retries for flakiness, and easy test authoring.
7

How would you design test cases for validating a login API?

Answer: Cover valid/invalid credentials, expired tokens, rate limits, lockouts, session handling, input validation, and security checks like SQL injection.
8

What is the difference between regression testing and retesting?

Answer: Retesting verifies a specific defect fix; regression testing checks that recent changes haven't broken existing functionality.
9

How do you handle dynamic elements in UI automation?

Answer: Use resilient selectors (data-* attributes), wait strategies, relative locators, and abstract UI interactions to reduce brittleness.
10

What strategies do you use to ensure your automated tests are maintainable and scalable?

Answer: Modularize tests, use page objects, parameterize data, keep tests deterministic, and run tests in CI with parallelization and tagging.
11

How would you test an API that has rate limiting and throttling?

Answer: Simulate high request rates, validate rate-limit headers and responses, test client backoff behavior, and ensure graceful degradation and monitoring.
12

What is the role of assertions in automated testing?

Answer: Assertions check expected outcomes; use clear assertions for behavior, not implementation details, and provide informative failure messages.
13

How do you integrate automated tests into a CI/CD pipeline?

Answer: Run tests on merge/build triggers, separate fast unit tests from slower integration/e2e tests, report failures, and gate deployments on test success.
14

What are the advantages and disadvantages of using Selenium for web automation?

Answer: Advantages: cross-browser, widely supported; Disadvantages: brittle UI tests, slower, and requires maintenance; mitigate with good patterns and headless runs in CI.
15

How do you handle test data management in automation testing?

Answer: Use isolated test environments, seed data fixtures, use factories, clean up after tests, and avoid relying on shared mutable data.
16

What is the difference between black-box testing and white-box testing?

Answer: Black-box tests validate behavior without internal knowledge; white-box tests use internal code structure to design tests and cover paths.
17

How would you test a system that interacts with third-party services or APIs?

Answer: Use contract tests, mocks/stubs for external services, test failure modes (timeouts/errors), and run integration tests against sandbox environments.

Mid-Level Questions

1

Write test cases for a user signup flow.

Answer: Cover positive signups, invalid inputs, duplicate users, email verification, and error handling for downstream services.
2

How do you create reliable UI tests?

Answer: Use resilient selectors, avoid timing assumptions, wait for stable states, and isolate tests with seeded data and mocks.
3

Explain test data management strategies.

Answer: Use fixture factories, isolate datasets per test, clean up after tests, and use synthetic or masked production-like data for realism.
4

How to measure test coverage and its limitations?

Answer: Coverage tools measure lines/branches executed but don't guarantee correctness; focus on meaningful tests for behavior and edge cases.
5

How do you reduce flakiness in automated tests?

Answer: Stabilize environment, avoid shared state, use retries judiciously, improve waits, and mock external unstable services.
6

Design a smoke test suite for deployments.

Answer: Include critical path tests, quick assertions for health, basic end-to-end flows, and run automatically post-deploy to validate readiness.
7

Explain contract testing and when to use it.

Answer: Contract tests verify provider-consumer expectations, useful for microservices to prevent breaking changes without heavy integration tests.
8

How do you test for performance regressions?

Answer: Baseline performance metrics, run load tests, compare against thresholds, and automate periodic benchmarks in CI.
9

How to test error handling paths?

Answer: Inject faults, mock downstream failures, verify retries and fallback behavior, and ensure proper user-visible messages and logging.
10

How to prioritize test automation efforts?

Answer: Prioritize high-risk and high-value flows, repeatable manual tests, and areas with frequent regressions to maximize ROI.
11

How to integrate tests into CI pipelines?

Answer: Split tests by speed, run unit tests on every commit, run integration/e2e on PRs or nightly, and fail builds on critical test failures.
12

Explain mutation testing and its benefits.

Answer: Mutation testing injects changes to ensure tests catch defects; it measures test suite quality beyond coverage by validating assertions.
13

How to validate third-party integrations?

Answer: Use sandbox environments, contract tests, mock failures, and monitor production behavior with alerts for integration failures.
14

What are best practices for logging to support testing?

Answer: Add structured logs with correlation IDs, include enough context, and avoid sensitive data to enable test reproducibility and debugging.
15

How to design resilience tests?

Answer: Include failure injection, network partition tests, dependency latency simulation, and verify graceful degradation and recovery behavior.
16

How to test data pipelines end-to-end?

Answer: Use synthetic input streams, validate outputs against expected aggregates, and include checks for completeness and correctness at each stage.
17

How to design tests for concurrency issues?

Answer: Create stress tests that increase concurrency, use race detectors, run tests in CI with various timing and ordering, and analyze flakiness patterns.
18

How to measure test suite flakiness over time?

Answer: Track failure rates, re-run failed tests to determine flakiness, and prioritize stabilizing the most flaky tests.
19

How do you secure test environments?

Answer: Isolate test infra, restrict data access, mask production data, and manage secrets and credentials securely in test pipelines.
20

How to design a testability-first architecture?

Answer: Design for dependency injection, small testable units, clear interfaces, and provide hooks/mocks for external systems to enable fast reliable tests.

Senior-Level Questions

1

How do you build a test strategy for a complex product?

Answer: Define quality goals, map critical journeys, allocate testing across unit/integration/e2e/production monitoring, and measure against SLOs with risk-based prioritization.
2

Explain shift-left testing and how to implement it.

Answer: Move testing earlier by adding unit tests, static analysis, and developer-run checks in CI; embed testers in teams and automate feedback loops.
3

How do you quantify test ROI?

Answer: Measure defects caught pre-release, reduction in escaped defects, time saved in debugging, and business impact of prevented outages to assess ROI.
4

How to scale test automation across many teams?

Answer: Provide shared frameworks, templates, training, centralized CI resources, and enforce standards while enabling team autonomy for tests relevant to their domain.
5

Explain building reliability engineering into testing culture.

Answer: Integrate SRE principles, define SLIs/SLOs, test runbooks, and incorporate chaos and resilience tests into regular pipelines to validate reliability.
6

How do you design metrics to prove testing effectiveness?

Answer: Track escaped defect rate, test pass rates, flakiness, test execution time, and coverage of critical user journeys to show testing impact.
7

How to embed security testing into CI/CD?

Answer: Automate static analysis, dependency scanning, secret detection, and run dynamic security tests on staging with policy gates before deployment.
8

How to manage test data privacy at scale?

Answer: Mask or synthesize data, control access with RBAC, and audit usage; prefer synthetic data for public or shared environments.
9

How to evolve a legacy test suite to be maintainable?

Answer: Gradually refactor tests, remove brittle ones, add abstraction layers, modularize fixtures, and prioritize tests that provide the most value.
10

How to measure the quality of test automation code?

Answer: Use code reviews, linters, test coverage of test code, maintainability metrics, and enforce design patterns to keep automation reliable.
11

How would you build a cross-team testing center of excellence?

Answer: Create shared standards, reusable components, training, and a support model to help teams adopt best practices while tracking ROI and adoption metrics.
12

How to integrate production monitoring into test strategy?

Answer: Feed production telemetry into test planning, use production incidents to create regression tests, and monitor canary deployments closely.
13

How do you evaluate and reduce technical debt in test suites?

Answer: Identify flaky/slow tests, refactor or remove low-value tests, prioritize based on impact, and allocate time for maintenance in sprints.
14

How to ensure tests remain relevant as product evolves?

Answer: Review test coverage with product changes, adjust test cases for new behavior, and tie tests to acceptance criteria to keep them aligned.
15

Explain testing strategies for multi-cloud deployments.

Answer: Test across clouds using abstraction layers, validate infra-as-code in each provider, and include cross-region and failover scenarios in test plans.
16

How to design high-fidelity production-like test environments?

Answer: Use infrastructure-as-code to mirror production, subset data with anonymization, and provide deterministic test data to enable reliable end-to-end tests.
17

How to approach testing for data-intensive applications?

Answer: Use sampling of realistic data, validate aggregations, test scale and latency under realistic loads, and automate integrity checks across pipelines.
18

How do you design a phased rollout plan with testing controls?

Answer: Use feature flags, canary groups, monitoring gates, and staged rollouts with quick rollback criteria tied to health metrics.
19

How to use AI/ML to improve test automation?

Answer: Use ML to prioritize tests, detect flaky tests, generate tests from logs, and assist in failure triage and root cause suggestions.
20

How to build metrics and dashboards for test engineering?

Answer: Track test pass rates, flakiness, execution time, coverage of critical journeys, and define dashboards for stakeholders with alerts for regressions.
