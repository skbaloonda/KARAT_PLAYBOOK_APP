Practice Questions

Comprehensive set of practice questions for data engineering interviews, organized by experience level.

Junior Questions

1

Write a SQL query to select all customers from a table.

Answer: SELECT * FROM customers; (In production, specify columns and add filters/pagination.)
2

How do you join two tables in SQL?

Answer: Use JOIN clauses: SELECT ... FROM a JOIN b ON a.key = b.key; choose INNER/LEFT/RIGHT depending on desired rows.
3

Use Pandas to clean missing values in a dataset.

Answer: Use df.dropna() to remove or df.fillna(value) to impute; consider column types before choosing strategy.
4

Write a query to count orders per customer.

Answer: SELECT customer_id, COUNT(*) AS orders FROM orders GROUP BY customer_id;
5

Create an ER diagram for a library system.

Answer: Entities: Books, Authors, Members, Loans; relationships: Members->Loans->Books with FK constraints.
6

How do you filter data in SQL?

Answer: Use WHERE clauses: SELECT ... FROM table WHERE condition; combine with AND/OR and use indexed columns for performance.
7

Use NumPy to calculate the mean of a list.

Answer: import numpy as np; np.mean(my_list)
8

Write a query to group sales by product.

Answer: SELECT product_id, SUM(amount) FROM sales GROUP BY product_id;
9

How do you remove duplicates in Pandas?

Answer: Use df.drop_duplicates(subset=[...]) to remove duplicate rows based on selected columns.
10

Write a query to find the maximum value in a column.

Answer: SELECT MAX(column_name) FROM table;
11

How do you validate data types in Python?

Answer: Use isinstance() checks or Pandas dtypes and apply conversions with pd.to_numeric / pd.to_datetime and handle errors.
12

Write a query to sort results by date.

Answer: SELECT ... FROM table ORDER BY date_column ASC/DESC;
13

Use Pandas to merge two DataFrames.

Answer: pd.merge(df1, df2, on='key', how='inner') to join on key with chosen join type.
14

How do you document a data pipeline?

Answer: Document inputs/outputs, schedule, transformation steps, owners, and include diagrams and sample data; store in central docs.
15

Write a query to find customers with no orders.

Answer: Use LEFT JOIN and IS NULL: SELECT c.* FROM customers c LEFT JOIN orders o ON c.id=o.customer_id WHERE o.id IS NULL;
16

Use SQL to calculate total sales.

Answer: SELECT SUM(amount) FROM sales;
17

How do you create a simple ETL script in Python?

Answer: Read source (CSV/DB), transform using Pandas, write to target, add logging and error handling, and schedule via cron/Airflow.
18

Write a query to filter out negative values.

Answer: SELECT * FROM table WHERE column >= 0;
19

Use Pandas to create a summary table.

Answer: Use groupby and agg: df.groupby(['col']).agg({'val':'sum'}).reset_index()
20

How do you secure sensitive data in a CSV file?

Answer: Encrypt the file at rest, restrict access, remove or mask PII before sharing, and never commit to source control.

Mid-Level Questions

1

Design an ETL pipeline for daily batch processing.

Answer: Extract from sources, transform with idempotent steps, load to warehouse, schedule via orchestrator, include monitoring and retries.
2

How do you optimize Spark jobs?

Answer: Tune partitions, persist intermediate RDDs, avoid wide shuffles, use broadcast joins for small tables, and monitor stages with UI.
3

Explain idempotency in data pipelines.

Answer: Ensure operations can be retried without adverse effects using dedup keys, upserts, and idempotent sinks to handle partial failures.
4

How to handle late-arriving data?

Answer: Use event-time windows with watermarks, re-process windows when late data arrives, and store raw events to allow backfills.
5

Design a data validation framework.

Answer: Implement schema checks, row-level validations, anomaly detection, enforce via pipeline gates, and alert on failures with failure handling strategies.
6

How do you manage schema evolution?

Answer: Use schema registries, backward/forward compatible changes, and versioned transformations with migration paths for consumers.
7

Explain partitioning vs bucketing.

Answer: Partitioning splits data into directories for pruning; bucketing groups files by hash to optimize joins; use both based on query patterns.
8

How do you secure data pipelines?

Answer: Encrypt data in transit and at rest, enforce IAM, isolate networks, rotate credentials, and audit data access.
9

How to implement hot-path vs cold-path processing?

Answer: Hot path: streaming for low-latency needs; cold path: batch for heavy processing and aggregated analytics, route events accordingly and maintain consistency.
10

Explain change-data-capture (CDC) and use cases.

Answer: CDC captures DB changes (binlog/WAL), streams them for replication, analytics, and event-driven architectures, enabling near-real-time syncs and auditability.
11

How to design for idempotent writes to downstream systems?

Answer: Use unique message IDs, upsert semantics, deduplication in consumers, and transactional outbox patterns to guarantee single application of events.
12

How do you monitor pipeline health?

Answer: Track metrics (latency, throughput, error rate), alert on SLA breaches, expose job logs, and provide dashboards with historical trends.
13

What tools do you use for orchestration?

Answer: Use Airflow, Prefect, or Dagster for batch orchestration; use native streaming orchestrators for streaming workflows.
14

How to handle backpressure in streaming systems?

Answer: Use bounded buffers, rate limiting, reactive streams, partitioning, and scale consumers; apply retention or durable storage to avoid data loss.
15

Explain serialization formats (Avro/Parquet/JSON) trade-offs.

Answer: Avro/Parquet are compact and schema-based (better for analytics); JSON is human-readable but larger; choose based on read/write patterns and tooling.
16

How to implement unit and integration tests for pipelines?

Answer: Unit-test transforms, use small sample data, mock external systems, and run integration tests in sandbox environments with representative data sets.
17

How do you handle duplicate events?

Answer: Use deduplication keys, idempotent sinks, and consumer-side dedup tables or windowed dedupe logic to ensure correctness.
18

Design a scalable metadata store for pipelines.

Answer: Use a transactional store for metadata, expose APIs for lineage and schema, and optimize queries with indexes and caching for frequent lookups.
19

How do you plan for capacity and scaling?

Answer: Forecast load, run capacity tests, provision autoscaling policies, and monitor key metrics to adjust resources proactively.
20

How to implement data retention and deletion policies?

Answer: Tag data with retention metadata, schedule lifecycle jobs to delete/archive, and ensure compliance with legal and business requirements.

Senior-Level Questions

1

Design a global data pipeline with multi-region replication.

Answer: Replicate critical topics across regions, use geo-partitioning for locality, resolve conflicts with leader/follower or CRDTs, and optimize for compliance and latency.
2

How would you implement exactly-once processing?

Answer: Use idempotent sinks, transactional writes (Kafka transactions), and deduplication with unique keys and offsets to guarantee once semantics.
3

Design a data mesh governance model.

Answer: Federated ownership, standardized contracts, shared platform for discovery and compliance, and central governance for cross-cutting concerns.
4

Explain end-to-end testing strategies for pipelines at scale.

Answer: Use representative datasets, sandbox infra, black-box tests for pipeline outputs, and periodic canary runs with known inputs to validate correctness.
5

How to design a cost-efficient streaming platform?

Answer: Right-size retention, tiered storage, use compacted topics for changelogs, and balance throughput vs retention costs with monitoring and lifecycle policies.
6

How would you implement cross-team data contracts?

Answer: Define schemas in a registry, validate at producer/consumer boundaries, version changes with compatibility checks and migration windows.
7

Explain strategies for high-throughput ingestion.

Answer: Use partitioned ingestion, batch writes, backpressure-aware producers, and scale horizontally with parallel consumers and optimized serializers.
8

How do you design for regulatory compliance in pipelines?

Answer: Implement access controls, encryption, data retention policies, auditing, and data masking/anonymization as required by regulations.
9

How to handle schema drift across producers?

Answer: Enforce schema registry validation, notify consumers, support backward/forward compatibility, and provide tooling to migrate consumers.
10

How do you measure pipeline reliability?

Answer: Track SLOs (success rate, latency), monitor lag, error rates and data completeness, and run post-run validations and alerts.
11

Design a metadata-driven pipeline platform.

Answer: Store pipeline definitions and schemas in metadata store, use generic runners to interpret metadata, and provide lineage and observability APIs.
12

How to approach vendor selection for streaming platforms?

Answer: Evaluate scalability, durability, ecosystem, operational burden, cost, and compatibility with existing tooling and SLAs.
13

Explain strategies for reproducible data processing.

Answer: Version code and schemas, snapshot inputs, use deterministic transforms, and record pipeline parameters and environment for replayability.
14

How to handle cross-region regulatory data locality?

Answer: Partition data storage by locale, restrict processing and access per region, and replicate only allowed derived datasets across regions with consent.
15

How do you scale metadata services?

Answer: Use scalable databases with caching, partition indices, and optimize read-heavy paths while ensuring transactional integrity for writes.
16

How to design ingress gateways for high-volume data sources?

Answer: Use load balancers, buffer queues, rate limiting, and partitioning upfront; validate and reject malformed inputs early to protect downstream systems.
17

What are best practices for documenting pipeline contracts?

Answer: Include schema definitions, example payloads, SLAs, owners, and backward compatibility rules in a discoverable registry with version history.
18

How to approach large-scale reprocessing needs?

Answer: Use replayable raw event logs, plan batch capacity, isolate reprocessing infra, and ensure idempotency in sinks to avoid duplicates.
19

Explain balancing consistency and availability in pipelines.

Answer: Choose consistency per use case; favor eventual consistency for scalability with compensating logic, and use strong consistency for critical transactional paths.
20

How to build a team to run and evolve data platforms?

Answer: Hire platform engineers, define SRE-like ops for pipelines, enable self-serve for data teams, and invest in documentation, onboarding, and observability tooling.

Mid-Level – 3 Mock Interview Scripts

Scenario 1: SQL Query for Top Customers
Interviewer:
Write a SQL query to find the top 5 customers by sales.
Candidate:
SELECT customer_id, SUM(sales) AS total_sales FROM orders GROUP BY customer_id ORDER BY total_sales DESC LIMIT 5;
Interviewer:
How would you handle ties?
Candidate:
Use RANK() window function to rank customers and filter where rank <= 5.
Scenario 2: Data Cleaning in ETL
Interviewer:
How would you clean and preprocess a messy dataset?
Candidate:
Remove duplicates, handle missing values, standardize formats, and validate data types using SQL and Python (Pandas).
Interviewer:
How do you automate this?
Candidate:
Build ETL scripts with scheduled jobs and error logging.
Scenario 3: Dashboard Design
Interviewer:
Design a dashboard for product performance.
Candidate:
Include KPIs like sales, returns, and ratings. Use filters for time and category. Visualize with bar charts and trend lines.
Interviewer:
How do you ensure data accuracy?
Candidate:
Validate source data and refresh dashboards on schedule.

Senior-Level – 3 Mock Interview Scripts

Scenario 1: Advanced SQL for Running Totals
Interviewer:
Write a SQL query using window functions for running totals.
Candidate:
SELECT date, sales, SUM(sales) OVER (ORDER BY date) AS running_total FROM sales_data;
Interviewer:
How do you optimize this for large datasets?
Candidate:
Use partitioning and indexing on the date column.

Interviewer:
Architect a scalable ETL pipeline for multi-source data.
Candidate:
Use modular ETL steps (extract, transform, load), schedule with Airflow, and store intermediate results in staging tables.
Interviewer:
How do you handle schema changes?
Candidate:
Implement schema versioning and automated migration scripts.
Scenario 3: Data Governance and Compliance
Interviewer:
Design a data governance framework for analytics.
Candidate:
Define data ownership, access policies, and audit trails. Use tools for lineage tracking and compliance monitoring.
Interviewer:
How do you ensure GDPR compliance?
Candidate:
Mask PII, log data access, and enable user data deletion.
